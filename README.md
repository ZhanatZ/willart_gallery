# Дизайн-проект Willart

Сайт-галерея рендеров квартиры. Одностраничный, адаптивный, минималистичный.

Работает без backend — просто статические файлы, которые можно опубликовать на GitHub Pages или любом другом хостинге.

---

## Быстрый старт

### 1. Установка

```bash
npm install
```

### 2. Запуск для разработки

```bash
npm run dev
```

Откроется в браузере по адресу `http://localhost:5173/willart_gallery/`.

### 3. Сборка для публикации

```bash
npm run build
```

Результат — папка `dist/`, готовая к размещению на хостинге.

### 4. Публикация на GitHub Pages

```bash
npm run deploy
```

После этого сайт доступен по адресу:
`https://zhanatz.github.io/willart_gallery/`

## Как добавить новые изображения

### Шаг 1. Положить файлы в нужную папку

Все изображения лежат в `public/images/`. Каждая комната — отдельная папка:

```
public/images/
├── kabinet/full/         ← Кабинет
├── koridor/full/         ← Коридор
├── kukhnya-gostinaya/full/  ← Кухня-гостиная
├── spalnya/full/         ← Спальня
└── sanuzel/full/         ← Санузел
```

Просто скопируйте файлы `.jpg` в папку `full/` нужной комнаты.

### Шаг 2. Сгенерировать превью

```bash
npm run thumbs
```

Скрипт автоматически создаст уменьшенные копии (600px ширина) в папке `thumb/` каждой комнаты. Уже существующие превью не перезаписываются.

### Шаг 3. Добавить файлы в конфиг

Откройте файл `src/config/gallery.config.ts` и добавьте имена файлов в массив `images` нужной комнаты:

```typescript
{
  id: 'kabinet',
  title: 'Кабинет',
  images: [
    '01.jpg', '02.jpg', '03.jpg',
    '04.jpg',  // ← добавленный файл
  ],
},
```

**Порядок в массиве = порядок на сайте.** Чтобы поменять порядок — просто переставьте элементы.

### Шаг 4. Проверить и опубликовать

```bash
npm run dev       # проверить локально
npm run deploy    # опубликовать
```

## Как изменить обложку (hero image)

Обложка — это фото на стартовом экране.

1. Положите файл в `public/images/hero/`
2. В `src/config/gallery.config.ts` измените путь:

```typescript
project: {
  title: 'Дизайн-проект Willart',
  description: '...',
  cover: 'images/hero/название_файла.jpg',
},
```

---

## Как изменить список комнат

В файле `src/config/gallery.config.ts` — массив `rooms`.

**Добавить комнату:**

1. Создайте папки `public/images/новая-комната/full/` и `public/images/новая-комната/thumb/`
2. Положите изображения в `full/`
3. Запустите `npm run thumbs`
4. Добавьте комнату в конфиг:

```typescript
rooms: [
  // ...существующие комнаты
  {
    id: 'novaya-komnata',
    title: 'Новая комната',
    images: ['01.jpg', '02.jpg'],
  },
],
```

**Удалить комнату** — просто удалите её из массива `rooms`.

**Изменить порядок комнат** — переставьте объекты в массиве.

## Структура проекта

```
willart_gallery/
├── public/images/          ← все изображения (не проходят через сборщик)
│   ├── hero/               ← обложка
│   ├── kabinet/full/       ← полноразмерные

```

|---------|-----------|
| `npm run dev` | Запуск для разработки (с горячей перезагрузкой) |

## Перенос на другой хостинг

Собранный проект (`dist/`) — обычные статические файлы. Можно разместить на:

- GitHub Pages (настроено)
- Netlify — просто перетащить папку `dist/`
- Vercel — подключить репозиторий
- Любой хостинг с поддержкой статики

При размещении в корне домена (не в подпапке) измените `base` в `vite.config.ts`:

```typescript
export default defineConfig({
  base: '/',   // ← корень домена
  // ...
});
```
