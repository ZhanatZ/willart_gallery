# Техническое задание  
## Проект: сайт-галерея рендеров квартиры  
## Название проекта: Дизайн-проект Willart

---

## 1. Общее описание проекта

Нужно разработать одностраничный адаптивный веб-сайт для публикации и просмотра реалистичных рендеров квартиры.

Сайт предназначен для личного использования и для передачи публичной ссылки строительной бригаде, чтобы можно было быстро сверяться с визуализациями по комнатам.

Основная задача сайта — удобный просмотр рендеров, сгруппированных по помещениям, без лишнего функционала.

Сайт должен быть визуально аккуратным, минималистичным, стильным и производить впечатление finished production-проекта, а не учебного шаблона.

Сайт не предполагает:
- формы обратной связи
- контакты
- SEO-фокус
- мультиязычность
- аналитику
- личный кабинет
- backend-часть
- пароли и скрытые разделы

---

## 2. Цель сайта

Сайт должен решать следующие задачи:

- показывать рендеры квартиры в удобном виде
- позволять быстро переключаться между комнатами
- открывать изображения в полноэкэкранном режиме
- корректно работать на мобильных устройствах и десктопе
- позволять владельцу проекта удобно добавлять изображения по комнатам
- быть пригодным для публикации по публичной ссылке
- быть легко переносимым между хостингами

---

## 3. Целевая аудитория

- владелец проекта
- строительная бригада
- ограниченный круг пользователей, которым будет передана публичная ссылка

---

## 4. Контент сайта

На сайте размещаются только рендеры квартиры.

### На старте:
- около 30 изображений
- изображения уже готовы
- изображения распределяются по комнатам

### Комнаты:
- Кабинет
- Коридор
- Кухня-гостиная
- Спальня
- Санузел

### Требования к контенту:
- у изображений не нужны подписи
- у разделов должны быть названия комнат
- порядок изображений внутри каждой комнаты должен задаваться вручную
- изображения должны храниться по папкам комнат
- должен быть отдельный конфиг-файл, где задаются порядок, названия и обложка проекта

---

## 5. Формат сайта и структура интерфейса

Сайт должен быть одностраничным.

### Структура страницы:

### 1. Hero-блок / стартовый экран
- крупное фото одной из комнат
- название проекта: Дизайн-проект Willart
- короткое описание проекта
- визуально чистый и спокойный первый экран

### 2. Блок переключения комнат
- под стартовым экраном
- в виде кнопок-плашек по комнатам
- активная комната визуально выделяется
- переключение между комнатами должно быть быстрым и понятным

### 3. Галерея выбранной комнаты
- заголовок комнаты обязателен
- изображения отображаются в виде сетки превью
- при клике открывается fullscreen/lightbox

### Footer
- не нужен

---

## 6. Визуальный стиль

### Общее направление:
- минималистичный
- стильный
- авторский
- без лишнего декора
- без тяжелых анимаций
- без “вау-эффектов” ради эффекта

### Цветовая концепция

Интерфейс должен быть:
- светлым и нейтральным
- не спорить с рендерами
- с деликатными акцентами в цветах интерьера

### Ориентиры по палитре:
- теплый светлый фон
- мягкие бежевые/молочные оттенки
- приглушенный оливковый как акцент
- возможны деликатные пыльно-розовые или теплые серо-бежевые нюансы
- черный/графитовый текст и элементы управления

### Визуальные принципы:
- много воздуха
- аккуратные отступы
- ровная типографика
- крупные удобные элементы на мобильном
- современная спокойная подача
- сайт должен выглядеть как реальный готовый продукт

---

## 7. Функциональные требования

### 7.1. Стартовый экран

На первом экране должны быть:
- название проекта
- краткое текстовое описание
- одно крупное изображение-обложка

### 7.2. Переключение по комнатам

Должен быть блок с кнопками комнат:
- Кабинет
- Коридор
- Кухня-гостиная
- Спальня
- Санузел

Требования:
- активная вкладка заметно выделяется
- переключение происходит без перезагрузки страницы
- по умолчанию открывается первая комната из конфига
- названия комнат отображаются на русском языке

### 7.3. Галерея

Требования к галерее:
- отображение изображений в виде сетки
- на мобильных — 2 колонки
- на десктопе — 3 или 4 колонки в зависимости от ширины экрана
- порядок изображений задается вручную
- изображения должны подгружаться оптимально
- превью должны быть аккуратно кадрированы и визуально единообразны

### 7.4. Fullscreen / Lightbox

При нажатии на изображение оно открывается в полноэкранном режиме.

Обязательные элементы:
- стрелка назад
- стрелка вперед
- свайп на телефоне
- счетчик вида: 3 из 8
- кнопка закрытия
- затемнение фона
- возможность листать изображения внутри выбранной комнаты
- корректная работа с клавиатуры на десктопе: Esc, стрелки влево/вправо

### 7.5. Управление контентом

Контент должен быть организован так, чтобы было удобно:
- загрузить изображения по папкам комнат
- задавать порядок показа
- менять стартовую обложку
- менять список комнат
- не переписывать код компонентов при добавлении изображений

Рекомендуемый принцип:
- изображения лежат в папках по комнатам
- есть единый конфиг-файл с описанием структуры проекта

## 8. Нефункциональные требования

### 8.1. Адаптивность

Сайт должен корректно работать на:
- мобильных устройствах
- планшетах
- ноутбуках
- больших экранах

### 8.2. Приоритет мобильной версии

Мобильная версия обязательна.

Особый приоритет:
- удобные кнопки
- понятное переключение между комнатами
- хорошее отображение сетки
- удобный fullscreen со свайпом

### 8.3. Производительность

Нужно предусмотреть:
- оптимизацию изображений
- lazy loading
- использование превью в сетке
- открытие более качественной версии в fullscreen
- отсутствие тяжелых библиотек без необходимости

### 8.4. Архитектура

Проект должен быть:
- чисто организован
- удобен для сопровождения
- пригоден для деплоя на GitHub Pages
- переносим на другой статический хостинг
- без привязки к сложной серверной инфраструктуре

### 8.5. Качество кода

Требования:
- читаемая структура
- переиспользуемые компоненты
- минимум хардкода
- данные вынесены в конфиг
- понятные имена файлов и переменных
- чистые стили
- отсутствие лишнего функционала

- пригоден для GitHub Pages
